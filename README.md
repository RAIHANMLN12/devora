# Devora

> **Scan, document, and test any API repository** — one CLI tool to document and test any API codebase.

[![CI](https://github.com/RAIHANMLN12/devora/actions/workflows/ci.yml/badge.svg)](https://github.com/RAIHANMLN12/devora/actions/workflows/ci.yml)
[![npm version](https://img.shields.io/npm/v/devora-cli.svg)](https://www.npmjs.com/package/devora-cli)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Devora automatically generates beautiful, interactive API documentation from your source code — no annotations, no config, no setup. It scans your routes, infers schemas, and serves a real-time documentation UI with an interactive testing sandbox.

## Quick Start

```bash
# Install globally
npm install -g devora-cli

# Or use directly via npx
npx devora-cli

# Scan your project
cd my-api
devora scan .

# View documentation
devora docs --open

# Interactive API testing
devora sandbox --mock
```

## Features

- **Zero-config scanning** — detects Express.js, Fastify, FastAPI, and Flask automatically
- **Schema inference** — understands `res.json()`, `req.body`, path params, and query params
- **Interactive docs UI** — searchable endpoint browser, schema viewer, dark/light mode
- **API sandbox** — test endpoints with mock responses or proxy to your real server
- **LLM enrichment** — optional AI-powered descriptions and examples (OpenAI, Anthropic, Ollama)
- **Watch mode** — auto-regenerate docs when source files change
- **Static export** — generate self-contained HTML documentation
- **Offline-first** — core scanning works without internet
- **Privacy-first** — no telemetry, no data leaves your machine

## Commands

| Command | Description |
|---------|-------------|
| `devora init` | Analyze repo and generate `.devora/config` |
| `devora scan .` | Scan source files and generate OpenAPI spec |
| `devora scan . --serve` | Scan and immediately serve docs |
| `devora scan . --llm` | Force LLM enrichment |
| `devora docs --open` | View API documentation in browser |
| `devora docs --export html` | Export as self-contained HTML |
| `devora sandbox` | Start interactive API testing |
| `devora sandbox --mock` | Use mock responses instead of a real server |
| `devora serve` | Run docs and sandbox together |
| `devora watch .` | Watch files and auto-regenerate |

## LLM Enrichment

Devora can optionally enrich your API documentation with AI-generated descriptions, parameter explanations, and example payloads.

```bash
# Set your API key
echo "OPENAI_API_KEY=sk-..." > .env

# Scan with LLM enrichment
devora scan .
```

Supports OpenAI, Anthropic, and local Ollama instances. Falls back gracefully if no key is found.

## Configuration

Devora uses a `.devora/config` file (auto-generated by `devora init`):

```yaml
# Devora Configuration
version: 1

scan:
  include:
    - "src/**/*.js"
    - "src/**/*.ts"
    - "routes/**/*.js"
  exclude:
    - "**/*.test.*"
    - "**/node_modules/**"
  framework: auto

llm:
  provider: auto
  model: auto
  temperature: 0.2
  enabled: true

docs:
  title: auto
  theme: dark
  port: 3456

sandbox:
  port: 3457
  proxyTarget: auto
  mock: false
```

## Output

Devora generates:

- `.devora/config` — Project configuration (YAML)
- `.devora/openapi.json` — OpenAPI 3.0.3 specification
- `.devora/docs/` — Exported static HTML documentation
